# 前端 API 變動串接規劃 — 2026-05

> 對應後端規格：[`plans/11-API-變動-2026-05.md`](./11-API-變動-2026-05.md)
> 影響範圍：`index.html`（主），`scent-journey.html`（不需動，history 結構未變）
> 目標：把前端從 2026-04 舊欄位（`deadline` / `nextOrderDate` / `order_created_at`）切換到 2026-05 新欄位（`t7Date` / `t8Date` / `nextChargeDate` / `isLocked` / `nextPerfume` / `currentPeriodNo`），並落實「鎖定期不隱藏 UI、但停用選香」的顯示邏輯。

---

## 一、核心顯示原則（決策）

| 原則 | 內容 |
|------|------|
| **時間語意只用 T+7 / T+8** | 不在 UI 顯示 `shippingDate`（避免「出貨進行中卻又能選香」的矛盾感）；只用 `t7Date`（出貨承諾日）與 `t8Date`（下期選香開放日） |
| **鎖定期 UI 不隱藏** | T 到 T+7 內，carousel、顧問 banner、保留區塊**全部保留**，只把選香按鈕 `disabled` 並顯示鎖定提示 |
| **截止日 = 下次扣款日** | 截止概念仍在，但對應到 `nextChargeDate`：扣款一觸發，下期就 promote 成本期、不能再改。文案：「下期選香截止日：{nextChargeDate}」 |
| **完全信任後端 `isLocked`** | 前端不再自行算週三 / 鎖定天數，按 `isLocked` 切 UI |
| **首期 = 一般開放期** | `currentPeriodNo === 1` 視同開放期（API 已保證 `isLocked=false`），但文案標示「首期選香（直接套用本期）」 |

## 二、欄位對照與處理

| 舊欄位（移除） | 新欄位（取代） | 前端處理 |
|----------------|-----------------|----------|
| `result.deadline` | `result.nextChargeDate` | 「下期選香截止日：{nextChargeDate}」（扣款一觸發就鎖死） |
| `result.nextOrderDate` | `result.nextChargeDate` | 同上；不再做日期解析 |
| `result.order_created_at` | `result.t7Date` / `result.t8Date` | 出貨/開放日直接拿後端字串，**移除** `parseSubscriptionDate` 與 `firstWednesdayOnOrAfter` 兩個 helper |
| `result.months === 1` 判斷首期 | `result.currentPeriodNo === 1` | 改用期數欄位 |
| — | `result.nextPerfume` | 新增；值 `"沿用本期"` 時 fallback 顯示 `currentPerfume` |
| — | `result.t7Date` | 鎖定期文案「{t7Date} 前送達」 |
| — | `result.t8Date` | 鎖定期文案「下期選香 {t8Date} 開放」 |

### 3.3 `renderScentBlock`（新函式，取代 `applyLockState`）

```js
function renderScentBlock({ currentPerfume, nextPerfume, currentPeriodNo, isLocked, t7Date, t8Date, nextChargeDate }) {
  isLockedGlobal = isLocked;  // 給 carousel click handler 用
  const isFirstPeriod = currentPeriodNo === 1;

  // ── 保留區塊「本月專屬保留」 ──
  // 顯示對象：鎖定期 → currentPerfume；開放期 → nextPerfume（"沿用本期" 時 fallback currentPerfume）
  const reservedName = isLocked
    ? currentPerfume
    : (nextPerfume && nextPerfume !== '沿用本期' ? nextPerfume : currentPerfume);
  setText('reservedPerfumeName', reservedName || '—');

  // ── 三段文案 ──
  if (isLocked) {
    setText('scentDeadlineNote', `本期出貨準備中，${formatMd(t7Date)} 前送達`);
    setText('shipDateNote',      `下期選香將於 ${formatMd(t8Date)} 開放，現在無法修改。`);
    setText('nextScentChannelNote', `下次扣款 ${formatMd(nextChargeDate)}。`);
  } else if (isFirstPeriod) {
    setText('scentDeadlineNote', '首期選香｜選擇後直接套用本期');
    setText('shipDateNote',      `本期出貨後將開始下期週期`);
    setText('nextScentChannelNote', `下次扣款日：${formatMd(nextChargeDate)}`);
  } else {
    const hint = (!nextPerfume || nextPerfume === '沿用本期')
      ? `不選將沿用本期：${currentPerfume}`
      : `下期已選：${nextPerfume}`;
    setText('scentDeadlineNote', `下期選香截止日：${formatMd(nextChargeDate)}（扣款後鎖定）`);
    setText('shipDateNote',      hint);
    setText('nextScentChannelNote', '');  // 開放期不重複提醒
  }

  // ── carousel 按鈕高亮 / 啟用狀態 ──
  // 高亮對象：首期 → currentPerfume；一般開放期 → nextPerfume（fallback currentPerfume）；鎖定期 → currentPerfume
  const highlightName = isFirstPeriod
    ? currentPerfume
    : (isLocked ? currentPerfume : (nextPerfume && nextPerfume !== '沿用本期' ? nextPerfume : currentPerfume));
  updateScentSelection(highlightName);

  // ── 鎖定提示 + disable 按鈕 ──
  const buttons = document.querySelectorAll('.scent-card-btn');
  buttons.forEach(btn => { btn.disabled = isLocked; });
  const lockNotice = document.getElementById('scentLockNotice');
  if (isLocked) {
    setText('scentLockNoticeText', `系統出貨準備中，下期選香將於 ${formatMd(t8Date)} 開放。`);
    lockNotice.classList.remove('hidden');
  } else {
    lockNotice.classList.add('hidden');
  }
}

// helpers
function setText(id, txt) { const el = document.getElementById(id); if (el) el.textContent = txt; }
function formatMd(isoDate) {
  // "2026-05-16" → "5 月 16 日"
  if (!isoDate) return '';
  const [, m, d] = isoDate.split('-');
  return `${parseInt(m)} 月 ${parseInt(d)} 日`;
}
```
