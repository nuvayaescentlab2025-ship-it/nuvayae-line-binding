## 一、查詢綁定狀態（取代原 GAS `doGet` + `userid` 參數）

### Request

```
GET {BASE_URL}?action=check_bound&userid={LINE_UID}
```

### Response（已綁定）

```json
{
  "status": 200,
  "isBound": true,
  "duration": "30 天 5 小時 12 分鐘",
  "months": 1,
  "subType": "month",
  "currentPerfume": "薰衣草",
  "currentPerfumeDate": "2026-04-01T10:30:00+08:00",
  "lastPerfume": "白茶",
  "deadline": "2026-04-28",
  "nextOrderDate": "2026-05-01",
  "isLocked": false
}
```

### Response（未綁定）

```json
{
  "status": 200,
  "isBound": false
}
```

### 欄位對照

| 欄位 | 型別 | 說明 |
|------|------|------|
| `isBound` | boolean | 是否已綁定有效訂閱 |
| `duration` | string | 訂閱持續時間（天/時/分） |
| `months` | number | 訂閱月數 |
| `subType` | `"month"` \| `"year"` | 訂閱方式 |
| `currentPerfume` | string | 當月選定香味，未選時為 `"尚未選香"` |
| `currentPerfumeDate` | string | 選香時間（ISO 格式），未選時為空字串 |
| `lastPerfume` | string | 上月選定香味；前端用於本月尚未選香時顯示「本月專屬保留」預設值，未選時為 `"尚未選香"` |
| `deadline` | string | 選香截止日（`yyyy-MM-dd`） |
| `nextOrderDate` | string | 下次出貨訂單日（`yyyy-MM-dd`） |
| `isLocked` | boolean | 目前是否已鎖定（出貨前 3 天內） |

### 前端範例

```javascript
const res = await fetch(
  `${BASE_URL}?action=check_bound&userid=${lineUid}`
);
const data = await res.json();

if (data.isBound) {
  // 顯示訂閱資訊
  console.log(`訂閱方式: ${data.subType}`);
  console.log(`當前香味: ${data.currentPerfume}`);
  console.log(`截止日: ${data.deadline}`);
  console.log(`已鎖定: ${data.isLocked}`);

  // 本月尚未選香時，以上月香氣作為預設保留顯示
  if (data.currentPerfume === '尚未選香' && data.lastPerfume && data.lastPerfume !== '尚未選香') {
    console.log(`本月專屬保留: ${data.lastPerfume}`);
  }
} else {
  // 導向綁定流程
}
```

## 三、綁定訂單（取代原 GAS `doPost` 預設行為）

### Request

```
POST {BASE_URL}
Content-Type: application/json
```

```json
{
  "action": "bind",
  "orderId": "ECPay20260401001",
  "phone": "0912345678",
  "lineUid": "U1234567890abcdef",
  "lineDisplayName": "Alice",
  "linePictureUrl": "https://profile.line-scdn.net/..."
}
```

### 參數對照（GAS → 新 API）

| GAS 參數 | 新 API 參數 | 必填 |
|-----------|-------------|------|
| `orderId` | `orderId` | 是 |
| `mobile` | `phone` | 否 |
| `userid` | `lineUid` | 是 |
| `displayName` | `lineDisplayName` | 否 |
| `pictureUrl` | `linePictureUrl` | 否 |
| — | `action` | 是（固定 `"bind"`） |

### Response 201（成功）

```json
{
  "status": 201,
  "message": "成功",
  "subscriptionType": "month",
  "currentPerfume": "薰衣草"
}
```

### 錯誤回應

| HTTP 狀態碼 | `message` | 說明 |
|-------------|-----------|------|
| 400 | `"缺少必要參數 (orderId 或 lineUid)"` | 參數不足 |
| 403 | `"訂單已超過一個月效期"` | 訂單太舊 |
| 403 | `"此訂單不包含訂閱方案，無法綁定"` | 非訂閱商品 |
| 404 | `"找不到對應訂單編號"` | 訂單不存在 |
| 409 | `"此 LINE 帳號已綁定訂閱"` | 重複綁定 |
| 409 | `"此訂單已被其他帳號綁定"` | 訂單已被佔用 |

### 前端範例

```javascript
const res = await fetch(BASE_URL, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    action: "bind",
    orderId: orderId,
    phone: mobile,
    lineUid: userId,
    lineDisplayName: displayName,
    linePictureUrl: pictureUrl,
  }),
});

const data = await res.json();

if (res.status === 201) {
  // 綁定成功
  alert(data.message);
} else {
  // 顯示錯誤
  alert(data.message);
}
```

## 四、選香 / 修改香味（取代原 GAS `doPost` + `action=select_perfume`）

### Request

```
POST {BASE_URL}
Content-Type: application/json
```

```json
{
  "action": "select_perfume",
  "lineUid": "U1234567890abcdef",
  "scentName": "薰衣草",
  "displayName": "Alice"
}
```

### 參數對照（GAS → 新 API）

| GAS 參數 | 新 API 參數 | 必填 |
|-----------|-------------|------|
| `userid` | `lineUid` | 是 |
| `perfumeName` | `scentName` | 是 |
| `displayName` | `displayName` | 否 |
| `action` | `action` | 是（固定 `"select_perfume"`） |

### Response 200（成功）

```json
{
  "status": 200,
  "message": "選香紀錄成功！",
  "targetMonth": "2026-05",
  "scentName": "薰衣草"
}
```

### 錯誤回應

| HTTP 狀態碼 | `message` | 說明 |
|-------------|-----------|------|
| 400 | `"缺少必要參數 (lineUid 或 scentName)"` | 參數不足 |
| 403 | `"找不到您的有效訂閱紀錄，請先完成綁定！"` | 未訂閱 |
| 403 | `"選香已鎖定！..."` | 出貨前 3 天鎖定 |

### 前端範例

```javascript
const res = await fetch(BASE_URL, {

    action: "select_perfume",
```

## 五、遷移檢查清單

前端更新時請逐項確認：

- [ ] 將所有 GAS URL 替換為 `https://kvqkjtyhbbcivwlexgcn.supabase.co/functions/v1/member`
- [ ] GET 請求加上 `action` 查詢參數（`check_bound` 或 `get_duration`）
- [ ] POST 綁定：body 改為 JSON 格式，新增 `action: "bind"`，參數名稱更新（`mobile` → `phone`、`userid` → `lineUid`、`displayName` → `lineDisplayName`、`pictureUrl` → `linePictureUrl`）
- [ ] POST 選香：參數名稱更新（`userid` → `lineUid`、`perfumeName` → `scentName`）
- [ ] 確認 `Content-Type: application/json` header 已設定
- [ ] 錯誤處理改用 HTTP 狀態碼判斷（不再只看 response body 的 `status` 欄位）
- [ ] 綁定成功判斷改為 `res.status === 201`（非 200）
- [ ] 測試各流程：未綁定查詢、綁定、已綁定查詢、選香、鎖定期選香

## 六、GAS vs 新 API 差異摘要

| 項目 | GAS（舊） | Supabase Edge Function（新） |
|------|-----------|-------------------------------|
| URL | Google Script Web App URL | `https://kvqkjtyhbbcivwlexgcn.supabase.co/functions/v1/member` |
| 請求格式 | URL 參數（form data） | JSON body（POST）/ URL 參數（GET） |
| 路由方式 | 依參數名稱隱式分流 | 明確 `action` 參數 |
| 錯誤回應 | 全部 HTTP 200 + body 內 status | HTTP 狀態碼（400/403/404/409）+ body |
| CORS | 無 | 已內建（`Access-Control-Allow-Origin: *`） |
| 認證 | 無 | 無（開放存取，與 GAS 一致） |
