# 10 — 氣味旅程前端串接調整

> 後端 `check_bound` 新增 `history` 欄位後，前端需配合調整的項目。

---

## 異動範圍

僅 `scent-journey.html` 一個檔案。

---

## 調整項目

### 1. API 端點更新（必要）

**現狀**：仍使用舊版 GAS 端點

```js
// scent-journey.html:122
const API_URL = 'https://script.google.com/macros/s/AKfycby.../exec';
```

**目標**：改為與 `index.html` 一致的 Supabase Edge Function 端點

```js
const API_URL = 'https://kvqkjtyhbbcivwlexgcn.supabase.co/functions/v1/member';
```

同時更新 `loadJourney()` 的 fetch URL，加上 `action=check_bound`：

```js
// 現行（scent-journey.html:217）
const res = await fetch(`${API_URL}?userid=${encodeURIComponent(userId)}`);

// 改為
const res = await fetch(`${API_URL}?action=check_bound&userid=${encodeURIComponent(userId)}`);
```

### 2. 移除 Mock 資料（後端上線後）

**現狀**：`MOCK_HISTORY` 常數與 `buildItems()` 的 fallback 邏輯作為設計稿預覽

```js
// scent-journey.html:140-144
const MOCK_HISTORY = [
    { year: 2026, month: 3, monthIndex: 3, perfumeName: '朝光', isCurrent: true },
    ...
];
```

**目標**：後端 `history` 欄位上線且驗證無誤後，移除 `MOCK_HISTORY` 常數，並簡化 `buildItems()`：

```js
function buildItems(result) {
    return result.history || [];
}
```

> 注意：此項應在後端上線並驗證後再執行，避免中間期間頁面空白。

### 3. 空 history 的 UX 處理（建議）

**現狀**：若 `history` 為空陣列，`buildItems()` 會 fallback 至 Mock。Mock 移除後，空陣列會導致時間軸完全空白。

**建議**：新增空狀態提示

```js
if (!items.length) {
    timeline.innerHTML = '<p class="status-msg">尚無選香記錄。</p>';
    return;
}
```

## 不需調整的部分

| 項目 | 原因 |
|---|---|
| `SCENT_STYLES` 常數 | 後端確認 DB 存的香水名稱與前端 key 完全一致 |
| `SEASON_GIFTS` 常數 | 純前端展示邏輯，不依賴後端資料 |
| `renderItem()` 函式 | 已支援 `perfumeName: null` 的佔位卡片，無需修改 |
| `index.html` 跳轉邏輯 | 已正確帶入 `?userid=`（index.html:714），無需修改 |

---

## 執行順序

1. **立即可做**：調整項目 1（API 端點更新）— 不依賴後端，可先行部署
2. **後端上線後**：調整項目 2 + 3（移除 Mock + 空狀態處理）— 需確認 `history` 欄位正常回傳後再執行
